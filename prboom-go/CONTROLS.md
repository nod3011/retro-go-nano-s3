# Doom Input Controls (Nano S3)

Technical reference for the custom input mapping implemented in this port.

## 🕹 Standard Controls
| Hardware Button | Game Action |
| :--- | :--- |
| **D-Pad Up** | Move Forward |
| **D-Pad Down** | Move Backward |
| **D-Pad Left** | Turn Left |
| **D-Pad Right** | Turn Right |
| **A** | Use (Open Doors / Switches) |
| **B** | Sprint / Speed |
| **X / Y** | Fire Weapon |
| **L** | Previous Weapon |
| **R** | Next Weapon |

---

## 🛠 Menu Shortcuts (Hold 'Menu' + Button)
| Combination | Action |
| :--- | :--- |
| **Menu + Start** | **Quick Save** |
| **Menu + Select** | **Quick Load** |
| **Menu + B** | **Automap Toggle** |
| **Menu + Up** | **Toggle Always Run** |
| **Menu + Left** | **Decrease Gamma** (Brightness) |
| **Menu + Right** | **Increase Gamma** (Brightness) |
| **Menu (Release)** | **Esc (Doom Menu)** |

---

---

## 💡 Notes
- **Always Run**: When *Always Run* is toggled ON, the **B** button acts as a normal button (no speed inversion bug).
- **Modifier Interruption**: Holding any other button after pressing 'Menu' or 'Option' will prevent the system menu from opening on release.
